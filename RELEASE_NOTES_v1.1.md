# Release 1.1

**What's new**

- **Profile pages fixed** – The fix now works on profile pages (`/in/...`) as well as the feed. LinkedIn also probes via image loading on some pages; we now block that too.
- **All LinkedIn subdomains** – The extension runs on every LinkedIn URL (e.g. `www.linkedin.com`, `*.linkedin.com`), so no need to edit the manifest for your locale.
- **Troubleshooting in README** – Reload tab, check extension is enabled, and confirm site access if something doesn’t work.

**Upgrade:** Get the latest code, then reload the extension at `chrome://extensions/` and reload your LinkedIn tab. No new permissions.
